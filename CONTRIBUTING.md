# Contributing

## Ground Rules

- Keep proposals narrow and schema-focused.
- Prefer additive changes over breaking renames.
- Document compatibility impact in every schema-facing change.
- Do not widen the event model just to anticipate distant roadmap ideas.

## Pull Requests

Include:

- the event or field contract being changed
- the motivation for the change
- expected downstream impact on `carapace-js`, `carapace-detection-rules`, and `carapace-playground`

## Early Stage Expectation

During the first slice, issues and pull requests should stay tightly tied to the locked Carapace event shortlist.
